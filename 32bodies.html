bodies¶

This module provides a high level data structure, representing entities such as planets, spacecraft or asteroids.

Note

The constructor of Body (and all its subclasses) is actually a factory function and the type of the returned entity depends on the provided body_id.

Implementation Detail

Once created, entities are cached and treated as singletons. Because of this, they will not change if the underlying name <-> Id mapping is modified at runtime. At the moment there is no API to make changes to that mapping, but it’s still good to know.

class spiceminer.bodies.Asteroid(body_id)¶

Subclass of Body representing asteroids.

Asteroids are ephimeris objects with IDs > 200000.

class spiceminer.bodies.Barycenter(body_id)¶

Subclass of Body representing a barycenter of an ephimeris object and all of its satellites.

Barycenters are ephimeris objects with IDs between 0 and 9.

class spiceminer.bodies.Comet(body_id)¶

Subclass of Body representing comets.

Comets are ephimeris objects with IDs between 100000 and 200000.

class spiceminer.bodies.Instrument(body_id)¶

Subclass of Body representing instruments mounted on spacecraft (including rovers and their instruments).

Instruments are ephimeris objects with IDs between -1001 and -10000.

class spiceminer.bodies.Planet(body_id)¶

Subclass of Body representing planets.

Planets are ephimeris objects with IDs between 199 and 999 with pattern [1-9]99.

class spiceminer.bodies.Satellite(body_id)¶

Subclass of Body representing satellites (natural bodies orbiting a planet).

Satellites are ephimeris objects with IDs between 101 and 998 with pattern [1-9][0-9][1-8].

class spiceminer.bodies.Spacecraft(body_id)¶

Subclass of Body representing spacecraft.

Spacecraft are ephimeris objects with IDs between -1 and -999 or < -99999.
